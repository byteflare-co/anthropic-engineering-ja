# Built with Opus 4.6 Claude Code ハッカソン受賞者の紹介

先週、私たちは [*Built with Opus 4.7* バーチャル Claude Code ハッカソン](https://cerebralvalley.ai/e/built-with-4-7-hackathon)を発表し、[最新](https://www.anthropic.com/news/claude-opus-4-7)の Opus モデルで何かを作ってみようとコミュニティに呼びかけました。Opus 4.6 をテーマにした前回のハッカソンと同様に、Cerebral Valley と連携して 500 名の参加者を選出し、それぞれに 500 ドル分の API クレジットと Claude Code で開発する 1 週間を提供します。その後、Claude チームの審査員が 6 名の受賞者を選び、プロジェクト開発用の Claude API クレジットとして総額 10 万ドルの賞金を授与します。

このシリーズの初回となる Opus 4.6 ハッカソンの受賞者には、人身傷害専門の弁護士、循環器専門医、道路・インフラのスペシャリスト、エレクトロニックミュージシャン、そして 1 名のプロのソフトウェアエンジニアが含まれていました。彼らは住宅、ヘルスケア、インフラ、音楽、教育を改善するプロジェクトに取り組みました。そして 5 名の受賞者のうち 4 名は、プロの開発者ではありませんでした。

彼らのプロジェクトが、あなたが意義あるものを作るきっかけになれば幸いです。

## 第 1 位：[CrossBeam](https://www.youtube.com/watch?v=jHwBkFSvyk0)、Mike Brown

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69e452d10be28aa3eb8577be_image3.png)

画像提供：Mike Brown。

カリフォルニア州の住宅許可は初回申請での却下率が 90% を超え、平均 6 か月の遅延によって住宅所有者には 3 万ドルのコストがかかっています。問題の多くは官僚的なもので、署名の漏れ、法令番号の誤り、書類の不備などです。

「みんなカリフォルニアには住宅危機があると思っています。でも違うんです。あるのは許可の危機なんです」と、人身傷害専門の弁護士である Mike Brown は語ります。新しい住居を建てるのに必要な許可を取得するには、建設そのものよりも時間がかかることもあります。Mike のハッカソンプロジェクトである [CrossBeam](https://github.com/mikeOnBreeze/cc-crossbeam) は、AI を使ってカリフォルニアの許認可のボトルネックを打破しようとしています。

CrossBeam では、建築業者が設計図と補正指示書をツールにドラッグアンドドロップするだけで、並列に動くサブエージェントが書類を解析し、空間インデックスを構築し、個々の補正項目に専門のエージェントを割り当てます。20 分後には、承認のための具体的なアクションプランが建築業者の手元に届きます。一方、行政側では、CrossBeam によって自治体が提出された許可申請を一括処理し、補正指示書のドラフトを自動生成できます。

南カリフォルニアの都市 Buena Park は、2029 年までに 8,900 戸以上の住宅を許可する必要がある一方で、2024 年の実績はわずか約 120 戸でした。そこで CrossBeam の導入が検討されており、建築業者だけでなく、山積みの書類をレビューする担当者の許可プロセスも迅速化しようとしています。

「カリフォルニアの許可危機の両サイドを解決できれば」と Mike は言います。「カリフォルニアの住宅危機そのものを解決できるかもしれません」。

Mike は、Claude Code にプロンプトを与え、その後 Claude にテストを書かせるというワークフローで CrossBeam を構築しました。「このコンテストで自分が優勝したなんて、信じられません。私はコードを 1 行も書いていません」と彼は語ります。「コードを 1 行も読んでさえいないのです」。

[*CrossBeam を GitHub でチェックする。*](https://github.com/mikeOnBreeze/cc-crossbeam)

## 第 2 位：[Elisa](https://www.youtube.com/watch?v=rsUaz_QAK6o)、Jon McBee

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69e45316687854976b223003_image2.png)

画像提供：Jon McBee。

Jon McBee の 12 歳の娘が 7 年生の科学研究プロジェクトでマイクロコントローラをプログラムする必要が出てきたとき、彼はソフトウェアエンジニアとして日々使っているのと同じツール、Claude Code を娘にも使えるようにしたいと考えました。しかしターミナルインターフェースは中学生向けに設計されていません。そこで、彼女のためのインターフェースを自作したのです。

Elisa は、ユーザーがプリミティブ（ゴール、要件、エージェント、スキル、ルール、ポータル、デプロイメント）をつなぎ合わせてソフトウェアを設計できる、ブロックベースのビジュアル統合開発環境（IDE）で、裏側では AI が実際のコードを書いてくれます。ユーザーはビジュアル言語で仕様を書き、メタプランナーがそれをタスクグラフに分解し、あとはエージェントが処理します。組み込みのティーチングエンジンが、使われているプログラミング概念を年齢に応じた形で解説するので、ビルド作業がそのままレッスンになります。Jon の娘はこれを使って、1 行もコードを書かずにプロジェクト用のマイクロコントローラにコードを書き込みました。

Jon は Claude Code を使って、30 時間で Elisa を作り上げました。76 回のコミット、39,000 行以上のコード、1,500 を超えるテストを積み上げたのです。「私はシステムアーキテクチャを知っています。ハードウェアを統合する方法も、ソフトウェアを定義しテストする方法も知っています」と彼は言います。「Claude Code は、その知識のすべてを、わずか 6 日で出荷可能な製品へと変えるのを手伝ってくれました」。

教育関係者からは、Elisa を教室で使いたいという問い合わせが寄せられており、Jon は 2 位の賞金として受け取った 3 万ドルの Claude API クレジットで、その取り組みを資金面で支援しようとしています。彼は、ソフトウェア制作者は近いうちにソースコードを書くのを完全にやめ、代わりに明確に定義されたテストと仕様に基づいて仕事をするようになると考えています。*Elisa* はそのアイデアを、子ども向けのビジュアルインターフェースで包んだものです。「このプロジェクトを娘にちなんで名付けました」と Jon は言います。「なぜなら、彼女こそがこれを使うべき人だからです」。

[*Elisa を GitHub でチェックする。*](https://github.com/zoidbergclawd/elisa)

## 第 3 位：[PostVisit.ai](https://youtu.be/V29UCOii2jE)、Michał Nedoszytko

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69e453686a5b08ad802e98f2_image1.png)

画像提供：Michał Nedoszytko。

Michał Nedoszytko はブリュッセルを拠点とする循環器専門医で、医療の傍らで 20 年にわたってヘルスケアソフトウェアを開発してきました。前作の Previsit.AI は、ベルギー、ギリシャ、ポーランドで導入されている AI 患者問診システムです。しかしこの 2 年間、彼が本当に作りたかったのはその対となるプロダクト、つまり診察の「あと」に何が起こるかを支えるツールでした。「カテーテル室で何千件もの処置を行ってきました」と彼は言います。「でも本当の苦労は、患者が部屋を出たその瞬間から始まるのです」。

PostVisit は、診断を平易な言葉で説明し、診察記録や AI スクライブの書き起こしを分析し、科学的リソースや医師の監督下にある完全な医療記録から関連する臨床エビデンスを提示する、一連のツール群です。患者は自分自身のケアをよりクリアに把握でき、医師は診察と診察の間に患者がどう過ごしているかを可視化できます。このアシスタントは、プライバシー、セキュリティ、臨床のベストプラクティスを軸に設計されています。

PostVisit を作るために、Michał はハッカソン・ロードトリップに出かけ、ブリュッセルからサンフランシスコへ向かう道中で開発を進めました。「道の上こそ、最高のアイデアが生まれる場所なんです」と彼は言います。1 週間と数千マイル後、彼は何年も思い描いていたプロダクトを手にしました。「医療はエビデンスに基づいています」と彼は語ります。「そして今、医療記録、エビデンス、診察データを組み合わせることで、診察後に何が起こるかを患者が完全にコントロールし、理解できるようになるのです」。

[*詳しくは postvisit.ai で。*](http://postvisit.ai)

## "Keep Thinking" 賞：[TARA](https://www.youtube.com/watch?v=GFCrXehS1DE)、Kyeyune Kazibwe

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69e4550a09fb338e86755644_image4.png)

画像提供：Kyeyune Kazibwe。

ウガンダでは、対応すべき道路インフラのニーズが予算をはるかに上回っており、従来のフィージビリティスタディも役には立ちません。これには 100 万〜400 万ドルのコストがかかり、9〜14 か月を要し、しかも道路が実際に奉仕する地域社会を考慮せず経済予測に偏りがちだからです。当時ウガンダの運輸省に勤務していた Kyeyune Kazibwe は、そうした意思決定のあり方を変えるべく TARA を作りました。

TARA は、ドライブレコーダーで撮影した道路映像を、完全な投資評価レポートへと変換します。Opus 4.6 のビジョン機能を使ってフレームごとに分析し、路面状態、劣化パターン、そして歩行者、自転車、露店を含む沿道の活動を識別します。システムは道路を状態ごとのセクションに分割し、介入コストを自動入力し、NPV、キャッシュフロー予測、感度分析を含む完全な経済評価を生成します。さらに、周辺施設や走行中に特定された重点地域を加味して、その投資が実際に誰に恩恵をもたらすかを評価するエクイティスコアも出力します。

ハッカソンのために、Kyeyune はウガンダで現在建設中の Kira - Matugga Road で撮影した実際のドライブレコーダー映像をアップロードしました。「ワンクリックで、状態評価、経済分析、エクイティの所見、感度の解釈をすべて 1 つの文書にまとめた完全な PDF レポートが生成されます」と彼は言います。「以前は何週間もかかっていた作業です。TARA は 5 時間でこなします」。

[*TARA を GitHub でチェックする。*](https://github.com/Kye256/tara-transport-assessment)

## 特別賞 — Creative Exploration：[Conductr](https://www.youtube.com/watch?v=X6CqJoyj0kI)、Asep Bagja Priandana

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69e45536f6ba167d389c055f_image7.jpg)

画像提供：Asep Bagja Priandana。

Asep Bagja Priandana は、Claude をライブの仮想バンドメンバーに変えるべく Conductr を作りました。このブラウザベースの MIDI 楽器は、コントローラーで弾いたコードを聴き取り、演奏を分析して、ドラム、ベース、メロディ、ハーモニーの 4 トラックをリアルタイムに生成します。*"make it funky"* や *"build to a climax"* とタイプすれば、ジャムの途中でアレンジが変化します。

技術的な難しさは、音楽を途切れさせないことだったと彼は言います。C で書かれたエンジンを WebAssembly にコンパイルし、15 ミリ秒ごとにノートを生成することで、AI の判断が流れを途切れさせることなくアレンジを作り直せるようになっています。レイテンシは、Asep の言葉を借りれば「音楽的に見えない」レベルです。

Conductr は、約 4,800 行の JavaScript と WebAssembly で動作しています。リアルタイムにあなたの演奏を聴き、考え、共に奏でる楽器としては、非常に無駄のないビルドです。

[*Conductr を GitHub でチェックする。*](https://github.com/nanassound/conductr)
