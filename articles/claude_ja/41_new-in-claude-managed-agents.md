# Claude Managed Agents の新機能: ドリーミング、アウトカム、マルチエージェントオーケストレーション

本日、Claude Managed Agents の dreaming（ドリーミング）をリサーチプレビューとしてローンチします。ドリーミングは [メモリー](https://claude.com/blog/claude-managed-agents-memory) を拡張する機能で、過去のセッションを振り返ってパターンを見つけ出し、エージェントの自己改善を支援します。あわせて、outcomes（アウトカム）、マルチエージェントオーケストレーション、Webhook も Managed Agents で構築する開発者に向けて提供を開始します。これらのアップデートにより、エージェントは最小限のステアリングで複雑なタスクをこなせるようになります。

## **ドリーミングで自己改善するエージェントを作る**

[ドリーミング](https://platform.claude.com/docs/en/managed-agents/dreams) は、エージェントのセッションとメモリーストアを定期的に振り返り、パターンを抽出してメモリーをキュレーションし、エージェントが時間とともに改善していくようにするスケジュールド処理です。コントロールの度合いはあなたが決められます。ドリーミングにメモリーを自動更新させることもできますし、変更が反映される前に自分でレビューすることもできます。

![](https://cdn.prod.website-files.com/68a44d4040f98a4adf2207b6/69f8e9ad765c7eed52dcf468_Claude-Managed-Agents-Blog-Followup-Dreaming.png)

ドリーミングは、繰り返し発生する誤り、エージェントが収束していくワークフロー、チーム全体で共有される好みなど、単一のエージェントだけでは見えないパターンを浮かび上がらせます。さらに、メモリーが進化していく過程でも高シグナルな状態を保てるよう、構造を整え直します。これは長時間稼働の業務やマルチエージェントオーケストレーションで特に有用です。

メモリーとドリーミングが組み合わさることで、自己改善するエージェントのための堅牢なメモリーシステムが形成されます。メモリーは、各エージェントが*業務を進めるなかで*学んだ内容を捉えます。ドリーミングは、エージェント間で共有された学びを引き出し、最新の状態を保ちながら、*セッションとセッションの合間*にメモリーを洗練していきます。

ドリーミングは Claude Platform の Managed Agents で利用可能です。開発者は [こちらからアクセスを申請](https://claude.com/form/claude-managed-agents) できます。

## **より良いアウトカムを届ける**

[アウトカム](https://platform.claude.com/docs/en/managed-agents/define-outcomes) では、成功とはどういう状態かを記述するルーブリックを書き、エージェントがそれに向かって作業します。別のグレーダーが独立したコンテキストウィンドウのなかであなたの基準に照らして出力を評価するため、エージェントの推論に影響されることはありません。何かが基準を満たしていない場合、グレーダーは何を変える必要があるかをピンポイントで指摘し、エージェントはもう一度パスを実行します。

エージェントは「良い」とはどういう状態かを知っているときに最も良い仕事をします。たとえば、構造的なフレームワーク、プレゼンテーションの基準、満たすべき要件のセットなどです。アウトカムを使えば、エージェントはその基準に対して自分の仕事を照合し、出力が十分に良いものになるまで自己修正できます。一回ごとに人間がレビューする必要はありません。

アウトカムは、ディテールへの注意と網羅性が求められるタスクで特に有効です。コピーがブランドボイスに合っているか、デザインがビジュアルガイドラインに沿っているかといった主観的な品質に対しても機能します。テストでは、アウトカムによって標準的なプロンプティングループに比べてタスク成功率が最大 10 ポイント向上し、最も難しい問題で最大の伸びが見られました。アウトカムはファイル生成の品質も改善しており、社内ベンチマークでは docx でタスク成功率が +8.4%、pptx で +10.1% 向上しました。

アウトカムを定義してエージェントを走らせ、完了時に [Webhook](http://platform.claude.com/docs/en/managed-agents/webhooks) で通知を受け取る、という使い方も可能になりました。

## **複数のエージェントで複雑なタスクを処理する**

単一のエージェントでこなすには仕事が多すぎる場合、[マルチエージェントオーケストレーション](https://platform.claude.com/docs/en/managed-agents/multi-agent) を使うと、リードエージェントが仕事を分割し、それぞれを独自のモデル・プロンプト・ツールを持つスペシャリストに委任できます。たとえば、リードエージェントが調査を進めている間に、サブエージェントがデプロイ履歴、エラーログ、メトリクス、サポートチケットへと並列に手を伸ばしていく、といった具合です。

これらのスペシャリストは共有のファイルシステム上で並列に作業し、リードエージェント全体のコンテキストに貢献します。イベントは永続化され、すべてのエージェントが自分のしてきたことを覚えているため、リードエージェントはワークフローの途中で他のエージェントに状況を確認しに戻ることもできます。すべてのステップは [Claude Console](https://platform.claude.com/) で追跡できます。どのエージェントが何を、どの順序で、なぜ実行したのかが分かり、タスクがどう委任され実行されたかを完全に可視化できます。

## **チームが作っているもの**

各チームはドリーミング、アウトカム、マルチエージェントオーケストレーションを使って、自分の仕事を検証し、セッションをまたいで学習し、複雑な仕事を並列化するエージェントを送り出しています。

- [Harvey](https://www.harvey.ai/) は、長文ドラフトや文書作成のような複雑なリーガル業務をコーディネートするために Managed Agents を使っています。ドリーミングによって、彼らのエージェントはファイルタイプ別の回避策やツール固有のパターンを含めて、セッション間で学んだことを覚えていられるようになりました。テストでは完了率が約 6 倍に伸びています。
- Netflix のプラットフォームチームは、異なるソースから来る数百のビルドのログを処理する分析エージェントを構築しました。何千ものアプリケーションに影響する変更では、多くのアプリケーションをまたいで繰り返し起きる問題を見つけることが重要になります。マルチエージェントオーケストレーションによって、エージェントはバッチを並列に分析し、対応する価値のあるパターンだけを浮かび上がらせます。
- Every による [Spiral](http://writewithspiral.com/) は、新しい API と CLI の裏側で動くライティングエージェントを支えるために、マルチエージェントオーケストレーションとアウトカムを使っています。リードエージェントは [Haiku](https://www.anthropic.com/claude/haiku) で動作し、入ってくるリクエストをさばき、必要に応じてすばやくフォローアップの質問を投げかけ、ドラフト作成は [Opus](https://www.anthropic.com/claude/opus) で動作するサブエージェントに委任します。ユーザーが複数のドラフトをリクエストした場合、サブエージェントは並列に走ります。ライティングの品質は Spiral の中核的な価値なので、それを担保するためにアウトカムを使っています。各ドラフトは Every の編集原則とユーザーのボイスからなるルーブリックに照らして採点されます。どちらもメモリーから引っ張ってきます。基準を超えたドラフトのみが返されます。
- [Wisedocs](https://www.wisedocs.ai/blogs/building-managed-agents-for-document-verification) は、Managed Agents の上に文書品質チェックエージェントを構築し、アウトカムを使って各レビューを社内ガイドラインに照らして評価しています。レビューは現在、チームの基準と整合性を保ちながら 50% 高速に実行されています。
