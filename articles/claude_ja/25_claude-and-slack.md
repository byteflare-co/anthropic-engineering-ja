# Claude と Slack

***更新:*** *Slack コネクターが Claude Pro および Max サブスクライバー向けに利用可能になりました。（2026 年 1 月 26 日）*

Claude を Slack と連携する 2 つの方法をご紹介します。Claude を Slack ワークスペースに直接追加する方法と、Slack を Claude アプリに接続して、会話中に Claude が関連する Slack メッセージを検索・参照できるようにする方法です。

Claude と Slack が深く統合されることで、好きな場所で作業できるようになります。Slack の会話の中で直接 Claude の支援を受けたり、より深いリサーチプロジェクトのためにコンテキストが必要なときに Claude に Slack ワークスペースを検索させたりできます。重要なメッセージへの返信の下書き、関連するディスカッションを取り込んだミーティングの準備、共有ドキュメントの分析など、すべてを今お使いの場所で行えます。

### Claude を Slack に導入する

Slack で Claude を使うと、Web 検索、ドキュメント分析、既存のコネクションなど、すでにお使いの機能をすべて活用できます。つまり、ドキュメント、メール、その他接続済みのツールから、Slack の中で直接 Claude がコンテキストを取得できます。

Claude は、あなたがアクセス権を持つパブリックチャンネルとプライベートチャンネルのみを検索します。パブリックスレッドで Claude に支援を求めると、まずプライベートで返信を下書きします。チームに共有する前に確認、編集、または再生成でき、プロセス全体を通じてコントロールを維持できます。

Slack で Claude を使う方法は 3 つあります。

- **ダイレクトメッセージ**: @Claude とのプライベートな会話を開始し、リサーチ、ライティング、分析のための個別サポートを受けられます。
- **AI アシスタントパネル**: 任意のチャンネルから Slack の AI アシスタントヘッダーにある Claude アイコンをクリックして、進行中の会話を中断することなく Claude にアクセスできます。
- **スレッド参加**: @Claude をメンションすることで、任意のスレッドに Claude を追加できます。Claude は会話のコンテキストに基づいて返信を下書きし、共有前に確認できます。

### Slack を Claude に接続する

Slack を Claude に接続すると、Claude はワークスペースのチャンネル、ダイレクトメッセージ、共有ファイルを検索し、より良い回答やリサーチに必要なコンテキストを見つけます。

Claude は、関連する Slack のディスカッションを取り込んでミーティングの準備を支援したり、複数のチャンネルからプロジェクトの最新情報を収集したり、チームの会話に埋もれたアクションアイテムを見つけたりします。以下のような用途にご活用いただけます。

- **ミーティングの準備:** Slack のディスカッションを確認し、関連ドキュメントを取得し、最近の更新情報や主要な意思決定をまとめたブリーフィングを作成します
- **プロジェクトの調整:** チャンネル全体を検索して進捗状況を収集し、ブロッカーを特定し、進捗をまとめます
- **オンボーディングサポート:** チャンネル履歴、共有ドキュメント、関連するメールスレッドを確認することで、新しいチームメンバーが進行中のプロジェクトを理解できるよう支援します
- **ドキュメント作成:** 会話や関連する背景資料からコンテキストを取り込み、チームのディスカッションをドキュメントとして体系化します

この統合は、よりインテリジェントな AI 搭載ワークフローへの重要な一歩です。

「すべての企業が、AI エージェントと人間が手を取り合って働くエージェンティックエンタープライズへの道を歩んでいます。Anthropic と提携して Claude を Slack に統合し、Slack のコンテキストを Claude に取り込むことで、その旅を加速させ、最高水準の AI を業務の流れの中に直接もたらします」と、Salesforce の Slack 最高プロダクト責任者である Rob Seaman 氏は述べています。「Claude と Slack のユーザー間のギャップを埋めることで、仕事をより生産的に、より楽しく、そしてこれまで以上にインテリジェントにするシームレスな AI 体験を創出しています。」

### セキュリティとコントロール

Slack における Claude は、ユーザーの Claude アカウントと同じセキュリティ基準とプライバシー設定を維持します。ワークスペース管理者が組織全体のアプリ承認を管理し、個々のユーザーは既存の Claude アカウントで認証します。

Slack では、Claude はあなたが閲覧権限を持つチャンネルと会話にのみアクセスし、既存の Slack のすべての権限設定を尊重します。Slack の会話は、組織の保持ポリシーとセキュリティ設定に従います。

### はじめましょう

Slack における Claude は、有料 Slack プランをご利用のチーム向けに、本日より Slack Marketplace を通じて利用可能です。個々のユーザーが Claude アカウントで認証する前に、ワークスペース管理者が Claude アプリを承認する必要があります。

Slack コネクターは、Slack アプリで Claude をインストール済みの Claude Team および Enterprise プランのお客様にご利用いただけます。管理者はまず組織に対して[コネクター](https://claude.ai/redirect/claudedotcom.v1.14053bce-a886-4f55-af25-12f155a0e268/directory/597f662f-36de-437e-836e-5a81013cbfbe)を有効化する必要があり、その後ユーザーが Claude の設定の「コネクター」タブで設定できます。
