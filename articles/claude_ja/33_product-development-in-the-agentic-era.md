# エージェント時代のプロダクト開発

AI の時代にプロダクトマネージャーであることの皮肉のひとつは、私の仕事がかつてないほど人間らしく感じられるようになったことです。

プロダクトマネジメントの仕事は、常にクラフト（職人技）とアラインメント（調整）の混合でした。私のキャリアの大半において、1週間は後者で占められていました。すなわち、職能横断的なステークホルダーやチームメイトとのミーティング、ステータスレポート、エンジニアリングチームとのチケットバックログです。私は本能的かつ素早い意思決定をして、その後に擁護し、説得し、リソースを確保するための険しい戦いを繰り返すことに慣れていました。インパクトのあるプロダクトを出荷することは、生み出すというより取引のように感じられることが多かったのです。

Claude を使えば、アイデアにプレッシャーをかけて検証し、ワークフローを自動化し、行き詰まりを抜け出すことができます。私はようやく、ユーザーやチームと一緒に、常に最も重要だった部分——クラフト——に本当の時間を使えるようになりました。これらの新しいワークフローが私の日々を変えましたが、最も意味のある変化が起きたのは、[Claude Managed Agents](https://claude.com/blog/claude-managed-agents)（現在ベータ版）——クラウドホスト型エージェントを大規模に構築・デプロイするためのコンポーザブルな API スイート——の開発に取り組み始めたときでした。

この投稿では、Managed Agents が私のプロダクトマネージャーとしての働き方をどのように変えたか、そしてあなた自身のワークフローに取り入れられるいくつかのパターンを共有します。

## **プロダクト開発、過去と現在**

API 設計はかつてドキュメントとコメントスレッドの中に存在していましたが、[AI のエクスポネンシャル](https://claude.com/blog/product-management-on-the-ai-exponential)の上では、私たちは出荷するもの自体で構築します。ドキュメント上ではエレガントに読める仕様も、実際にそれに対して何かを構築しようとした最初の瞬間に崩れ去ることがあります。Claude Code を使えば、API 仕様のプリプロダクション版に対してエージェントの素描を描くことができ、午後のうちには本物のプロトタイプをエンドツーエンドで動かせます。

私たちは、自分たち自身のプリミティブで構築する中で学んだことに基づいて、API の抽象化や Claude Console の UX を何度も作り直しました——これらの変更は、たとえ数週間にわたるドキュメントレビューでも明らかにならなかったでしょうし、そうでなければユーザーフィードバックによって遅すぎるタイミングで判明していたはずです。私たちは今でも形を議論し、生の curl リクエストを実行して、ベアメタル体験に満足できているかを確認しています。しかし Claude Code は、基本的な「hello world」テストから機能するエージェントへと、同じ作業セッションの中で連れて行ってくれます。これらのエージェントを構築する中で、私たちのハーネスや API が次世代のモデルとタスクの進化に対してどのように柔軟に対応できるかを、より具体的に見通せるようになっています。

当初、これらのプロトタイプはプロダクトの形を整えるためだけのものでしたが、今では私の日々の仕事自体を進化させてもいます。私の PM としてのワークフローは、いまや私たちのプロダクト群の間で明確に分かれています。**Claude** と **Claude Cowork** は、オープンエンドなリサーチや発見——継続的な対話を望む、もやのかかった初期段階の探索——に使います。やるべき仕事の輪郭がはっきりしたら、**Claude Code** を使ってそのためのカスタムエージェントを書いて出荷します。エージェントは **Managed Agents** の上に構築します。

二段構えの効果が最大の解放となりました。一方では、自分のプロダクトに対して簡単に構築できることが、次に出荷できるものとして想像できる範囲の上限を引き上げます。もう一方では、プロダクトが世に出れば、同じ開発の筋肉が、これまでバックログで滞っていた運用業務のロングテールを自動化することを可能にしてくれます。

## **プロダクトマネージャーのための Managed Agents 活用例**

いまや私は、あらゆる「やるべき仕事」のために専用のエージェントを立ち上げています。構築は簡単です。Claude Code に Managed Agents の skill をロードし、欲しいものの簡単な素描を概説します。開発者の方は、最新版の Claude Code と組み込みの `claude-api skill` を使って Managed Agents で構築することもできます——「*start onboarding for managed agents in Claude API*」と Claude にプロンプトするだけで始められます。この skill が呼び出されると、Claude は道中の統合手順を説明しながらエージェントを構築するので、必要に応じて簡単に方向転換できます。

これらのエージェントの例には次のようなものがあります。

- **アダプション分析。** 私たちの内部データベースへの永続的なアクセスと、データスキーマを理解するための skill を備えたエージェントが、興味深い外れ値やパターンを浮かび上がらせるためのクエリを実行します。過去の実行の[メモリ](https://claude.com/blog/claude-managed-agents-memory)を持つことで、これまでの発見の上に積み上げ、視点を継続的に進化させることができます。
- **開発者センチメントのモニタリング。** 事前構築された [Web 検索ツール](https://platform.claude.com/docs/en/managed-agents/tools#available-tools)と注力領域に関するガイダンスを備えたエージェントが、特定のドメインリストをスキャンして最新の開発者フィードバックを集め、共通のテーマについて報告します。分析すべきコンテンツが膨大なので、リサーチを[複数のエージェント](https://platform.claude.com/docs/en/managed-agents/multi-agent)に並列で展開し、結果を待って、知見を統合します。
- **デモ構築。** デモ用の GitHub リポジトリ、ブランド資産、イベントデッキにアクセスできるエージェントが、事前構築されたテンプレートを、関連するオーディエンス（カンファレンスや顧客ミーティングなど）に合わせた洗練されたデモへと変えます。

Managed Agents のセッションはクラウドで実行されるので、私はその場を離れて、戻ってきたときには仕事が終わっているという状態を作れます。リリースごとに独自のクセがあるためにスケールできなかったプロセスも、Managed Agents を使えば簡単に自動化できるようになり、エージェントの実行のたびに退屈ではなく活力を感じられるようになりました。

## **クラフトを磨く余白を作る**

1年前なら、こうした類の仕事はすべて、職能横断的なスタッフィング要求や、混沌としたスプレッドシート、あるいは試すことすらできなかった中途半端な構想の中で、のろのろと進んでいたはずです。いまや Claude と Managed Agents によって、私は自分自身をスケールさせることができ、最もインパクトのあるプロダクトを開発するためにチームと協働することに時間を使えるようになりました。私の1日は、顧客と一緒に革新的なアイデアを生み出すこと、エンジニアリングのカウンターパートと一緒に曖昧でもやのかかった問題に深く入り込むこと、そしてフロンティアのプロダクトワークに本物の創造的エネルギーを注ぐことに広がっています。

もしあなたがプロダクトマネージャーで、まだエージェントを構築したことがないのであれば、今週そこから始めるのが私のおすすめです。あなたが常に存在してほしいと願ってきた実験やツールは、たった1つのプロンプトといくつかの API コールの先にあります。
