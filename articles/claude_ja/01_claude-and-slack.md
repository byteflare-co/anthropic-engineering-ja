# Claude と Slack

***アップデート:*** *Slack コネクタは現在、Claude Pro および Max のユーザーでも利用できるようになりました。(2026 年 1 月 26 日)*

私たちは、Claude と Slack を組み合わせる 2 つの方法を発表します。Slack ワークスペースに Claude を直接追加することもできますし、Slack を Claude アプリ側に接続して、会話中に関連する Slack メッセージを Claude が検索・参照できるようにすることもできます。

Claude と Slack を深く統合することで、あなたが作業したい場所で仕事が進められるようになります。Slack の会話の中で直接 Claude に手伝ってもらうことも、より深いリサーチプロジェクトのためにコンテキストが必要なときに Slack ワークスペースを Claude に検索させることもできます。重要なメッセージへの返信をドラフトしたり、ミーティング準備のために関連する議論を引き寄せたり、共有ドキュメントを分析したり——すでに仕事をしている場所ですべてが完結します。

### Claude を Slack に持ち込む

Slack 上で Claude を使うとき、あなたがすでに使っている機能——ウェブ検索、ドキュメント分析、既存のコネクション——がそのまま利用できます。つまり Claude は、あなたが接続したドキュメント、メール、その他あらゆるツールからコンテキストを引き出すことが、Slack の中で直接できるのです。

Claude が検索できる Slack チャンネルは、あなたがアクセス権を持つパブリックチャンネルとプライベートチャンネルのみです。パブリックスレッドで Claude に助けを求めると、まずはあなた宛てに非公開でドラフトを作成します。チームに共有する前に内容を確認・編集・再生成でき、プロセス全体のコントロールを保てます。

Slack で Claude を使う方法は 3 つあります。

- **ダイレクトメッセージ**: @Claude に向けてプライベートな会話を始めると、リサーチ・ライティング・分析について 1 対 1 でサポートを受けられます。
- **AI アシスタントパネル**: 任意のチャンネルで Slack の AI アシスタントヘッダにある Claude アイコンをクリックすると、進行中の会話を中断することなく Claude にアクセスできます。
- **スレッドへの参加**: 任意のスレッドで @Claude とメンションすれば、Claude を会話に加えられます。Claude は会話のコンテキストに基づいて返信をドラフトし、あなたは共有前にその内容を確認できます。

### Slack を Claude に接続する

Slack を Claude に接続すると、Claude はあなたのワークスペース内のチャンネル、ダイレクトメッセージ、共有ファイルを検索し、より良い回答やリサーチに必要なコンテキストを見つけ出します。

Claude は、ミーティングに関連する Slack の議論を引き寄せる、複数チャンネルからプロジェクトの最新状況をまとめる、チーム会話の中に埋もれたアクションアイテムを探し出す、といった準備作業を手伝ってくれます。たとえば次のような用途に使えます。

- **ミーティング準備**: Slack の議論を確認し、関連するドキュメントを引き寄せ、最新のアップデートと主要な決定事項をまとめたブリーフィングを作成。
- **プロジェクト調整**: チャンネル横断で検索し、ステータスアップデートを集め、ブロッカーを特定し、進捗を要約。
- **オンボーディング支援**: チャンネル履歴、共有ドキュメント、関連メールスレッドを確認し、新しいチームメンバーが進行中のプロジェクトを理解できるよう手伝う。
- **ドキュメント作成**: チームの議論をドキュメント化し、会話や関連する背景資料からコンテキストを引き出す。

この統合は、よりインテリジェントで AI 駆動のワークフローへの重要な一歩を表しています。

「あらゆる企業は、AI エージェントが人間と手を取り合って働くエージェント型エンタープライズへと向かっています。Anthropic と提携して Claude を Slack に、Slack のコンテキストを Claude に統合することで、その歩みを加速します——クラス最高の AI を仕事の流れの中に直接持ち込むのです」と、Salesforce の Slack 最高製品責任者 Rob Seaman は述べています。「Claude と Slack ユーザーの間のギャップを埋めることで、私たちはシームレスな AI 体験を作り出しています。これにより仕事は、これまで以上に生産的で、楽しく、そしてインテリジェントなものになります」

### セキュリティとコントロール

Slack 上の Claude は、ユーザーの Claude アカウントと同じセキュリティ基準とプライバシー設定を維持します。ワークスペース管理者は組織全体でのアプリ承認をコントロールし、個々のユーザーは既存の Claude アカウントで認証します。

Slack 上で Claude がアクセスできるのは、あなたに閲覧権限があるチャンネルと会話のみです——既存の Slack 権限をすべて尊重します。Slack の会話は、あなたの組織の保持ポリシーとセキュリティ設定に従います。

### 始め方

Slack 上の Claude は、有料 Slack プランのチーム向けに Slack マーケットプレイスを通じて本日から利用可能です。個々のユーザーが Claude アカウントで認証できるようになる前に、ワークスペース管理者がまず Claude アプリを承認する必要があります。

Slack コネクタは、Slack アプリに Claude をインストールした Claude Team および Enterprise プランのお客様に提供されます。管理者はまず組織向けに[コネクタ](https://claude.ai/redirect/claudedotcom.v1.5daf1bab-9027-4896-a175-564a24340e1e/directory/597f662f-36de-437e-836e-5a81013cbfbe)を有効化する必要があり、その後ユーザーが自分の Claude 設定の「Connectors」タブから設定できるようになります。
