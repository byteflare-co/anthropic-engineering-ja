# 非技術系のプロジェクトマネージャーが Claude Code でストレスマネジメントアプリを 6 週間で開発・リリースした方法

*シリーズ **Day zero: founder stories** では、世界で最も興味深く、刺激的なスタートアップの作り手たちを紹介しています。本記事では、Claude Opus 4.6 ハッカソンの受賞者であり Respiro の作者である* Kostiantyn Vlasenko に焦点を当てます。

|  |  |
| --- | --- |
| 創業者 | Kostiantyn Vlasenko |
| 国 | ウクライナ |
| スタートアップ | Respiro |

このインタビューのビデオ通話に参加する 15 分前、Kostiantyn Vlasenko は緊張していました。何を話すかを考えるのに気を取られすぎて、実は自分自身の不安に気づいていなかったのですが、彼のスマートフォンは気づいていました。

「アプリがそれを検知してこう知らせてくれたんです。*『おい、ちょっとボックスブリージングのショートセッションをやってみないか。ボックスブリージングは前にも君のストレスを和らげるのに役立ったよね』*とね」と彼は語ります。

そのアプリ [Respiro](https://apps.apple.com/us/app/respiro-stress-relief/id6758023997) は、Vlasenko が作ったものです。プロジェクトマネージャーとして 10 年のキャリアを持つ彼ですが、2026 年 2 月に [Built With Opus 4.6 Claude Code Hackathon](https://cerebralvalley.ai/e/claude-code-hackathon?tab=guest-list) のためのプロジェクトを作ろうと決めるまで、コードを 1 行もリリースしたことがありませんでした。それから 1 か月後、Respiro (科学に裏打ちされたストレスマネジメントのための iOS アプリ) は Apple App Store で公開され、数百人のユーザーを抱え、Vlasenko 自身が構築・オーケストレーションする 15 以上の特化したサブエージェントを含むアーキテクチャの上で動いています。

これらすべてを彼は Claude Code を使い、72 時間で、チームもプログラミング経験もなしに成し遂げました。

## どのアプリも解決できなかった問題

[Mythical Games](https://mythicalgames.com/) のプロジェクトマネージャーとして、キーウを拠点とする Vlasenko は、ステークホルダーからのプレッシャー、チームのダイナミクス、そしてプロダクトリリースの容赦ないペースに 10 年間向き合ってきました。彼は呼吸法とマインドフルネスがストレスレベルを抑えるのに役立つことに気づきました。しかし、見つけられなかったのは、実際にストレスを感じている瞬間に介入してくれるストレスマネジメントツールでした。

「既存のアプリは、夜 10 時とかに『深呼吸しましょう』と通知してくれます。でも、私が*いまこの瞬間に*ストレスを感じていることは理解していなかったんです」と彼は説明します。アイディアが結晶化したのは、ウクライナ西部の山岳地帯で休暇を過ごしていたときでした。個人デバイスからのストレス信号をリアルタイムで検知し、最も必要とされる瞬間にガイド付きのマインドフルブリージングエクササイズで介入するアプリを作る、というものです。唯一の問題は、彼がアプリの作り方を知らなかったことです。

## エージェントを人として管理する

当時、Vlasenko は Claude を Jira の更新や Slack でのミーティングノートの投稿といった業務の自動化に使っており、Claude Code CLI を使い始めたばかりでした。「そのとき思ったんです。『よし、チームメンバーが Claude を使ってどうコードを書いているかは知っている。なら自分にもできるんじゃないか？』とね」と彼は語ります。

彼はすぐに、自分のプロジェクトマネジメントの経歴がコーディングスキルよりも重要であることに気づきました。「私には実際の人間をマネジメントしてきた経験がたくさんあります」と彼は言います。「それが IDE 内のエージェントをマネジメントするのと同じことだと気づいて、こんなに簡単なんだととても驚きました。」

Vlasenko は冗談めかして、最初のプロンプトは要するに *Hey Claude, ストレスを減らしてくれるアプリケーションをくれ* というものだったと語りますが、それでも最初の結果には驚かされたと言います。「『すごい、これは素晴らしい。頭の中だけのものじゃなくて、*いまこの瞬間に*スマホの中で動いているじゃないか！』とね。」

しかし Vlasenko はそこで止まりませんでした。彼は Claude を使ってエージェントシステムの仕組みや、自分のストレス検知ロジックに供給できる Apple API を調べ、続けて Claude にマルチエージェントアーキテクチャの構築を手伝わせました。TCA アーキテクトエージェント、Swift デベロッパーエージェント、Metal スペシャリスト、コードレビュアーなど、それぞれ異なるモジュールで並列に動作させたのです。

Claude はピボットも痛みなくしてくれました。React Native ベースの Respiro の初期 MVP が、Vlasenko が Android 端末を持っておらずテストできないという問題に直面したとき、彼は Claude Code を使ってアプリを Swift でゼロから書き直し、それを数時間で終えました。

最初のアイディアから App Store にリリースできる本格的なバージョンの Respiro まで、合計でちょうど 6 週間弱かかったと彼は言います。

## Claude Code のコードから App Store へ

アプリを作ることと、それをリリースすることは別の話でした。Vlasenko は Apple Developer Program を一度も使ったことがありませんでした。彼は時に複雑なそのプロセスを Claude にステップごとにガイドしてもらいました。

「行き詰まったら、スクリーンショットを撮って Claude に『ねえ、ここで何を押せばいい？』と聞けばよかったんです」と彼は語ります。同じアプローチで、ロギング用の Sentry や分析用の Amplitude といったサードパーティサービスも設定しました。「Claude に *サインインを完了したよ、ここに API キーがある* と伝えると、99% のケースで Claude は最初の試みでうまくやってくれました」と Vlasenko は言います。残り 1% については、「最初のプロンプトで自分が求めているものをもっと明確にして洗練させるだけでよかったんです。あと数回試せば、Claude は欲しい結果を返してくれます」と続けました。

実のところ、Vlasenko は [Claude のビジョン機能](https://platform.claude.com/docs/en/build-with-claude/vision) を最も過小評価されている機能だと考えています。「Claude にスクリーンショットを送って分析させると、何が見えているかを教えてくれます。そしてこれを使って、Claude にどんなに複雑な UX でもガイドさせられるんです」と彼は説明します。「例えば、Meta で API トークンを作らなければならなくて、彼らのインターフェースで完全に迷子になっていたんですが、Claude は本当に理解していて、必要な場所どこへでも案内してくれるんです。」

Respiro のマーケティングも Claude を活用した取り組みです。Vlasenko が分析機能を必要としたとき、Claude は Amplitude の基本 SDK の統合にとどまらず、フルのユーザーファネル、リテンションメトリクス、デイリーおよびマンスリーアクティブユーザーのトラッキングまで設定してくれました。コンテンツが必要になると、Claude は彼のブログ記事を書き始め、TikTok の作成も手伝うようになりました。Claude はまた、Vlasenko が思いつかなかったであろうグロース戦略も提案しました。Respiro をクライアントに勧めてくれそうな心理学者やマインドフルネスの専門家にアプローチする、というものです。彼はそれを試しました。うまくいきました。専門家たちはアプリを推薦し始めたのです。

## Founder mode を仕事に活かす

Vlasenko が Respiro の構築を通じて身につけたスキルは、サイドプロジェクトだけに活かされたわけではありません。彼は今や Mythical Games での日常業務の一環として、コードリリースをコミットし、機能を直接リリースしています。さらに、彼は Respiro を作る中で開発した開発プロセスの社内アドボケートとなり、自分の Claude フォルダとワークフローをエンジニアリングチームと共有しています。

「ほとんどのメンバーが戻ってきて *おい、このワークフローは今のものよりずっといいぞ！* と言ってくれたんです」と彼は語ります。その結果、Mythical には今や Claude を通じてのみ仕事をデリバリーする小さな社内チームがあります。とはいえ、関係者全員にとって完全に摩擦のない移行ではなかった、と Vlasenko は観察します。「コードのすべての行を完全にコントロールするマインドセットを切り替えるのは、一部のエンジニアにとっては難しいことです」と彼は言います。「私は深いプログラミングのバックグラウンドがないから簡単でしたし、自然な働き方だと感じられました。」

Vlasenko にとって Respiro を作った経験は、何が可能で何が不可能かの境界を消し去ってくれました。彼は今、音声ガイド付きのプラクティスやモーションエクササイズをアプリの次のリリースに組み込むべく、Claude が一緒に書いたロードマップに沿って作業を進めており、通常の業務を終えた後に 7〜8 時間のコーディングセッションをこなしています。

「Claude Code は私の新しい中毒だと言えますね」と彼は語りました。
